# AI-Assisted Project Lens — pauseo.app

Purpose:
Use pauseo.app as a practical case study for AI-assisted project management and PMI-CPMAI learning.

Non-goal:
This file does not introduce AI features into the app.

## How AI is used

- ChatGPT: product strategy, review, risk analysis.
- Claude Code: controlled implementation assistant.
- Human owner: final product and code decisions.

## Governance rules

- One backlog item per commit.
- Plan before code.
- Human approval before code changes.
- MVP scope has priority over AI tool suggestions.
- No AI-generated medical advice.
- No user-facing AI coach in v0.1.

## Learning mapping

- Risk: scope creep, medical claims, privacy, AI hallucination.
- Governance: clear repo docs and decision boundaries.
- Lifecycle: small iterative delivery.
- Validation: product decisions tied to user friction and evidence.
- Responsible AI: AI assists development, not end-user health decisions.

## Reflection after each commit

- What risk did this reduce or introduce?
- Did this increase user value?
- Did this increase cognitive load?
- Did Claude Code stay within scope?
- Is anything now tempting but still out of scope?
