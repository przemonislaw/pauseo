# UX Principles — pauseo.app

## UX thesis

The user may open pauseo.app in a moment of craving, stress, shame, frustration, boredom, or low self-control.

Therefore the app must minimize cognitive load.

The product should feel like:

> “One calm hand on your shoulder. One small action. No lecture.”

## Core UX principles

### 1. One primary action per screen

Every screen should have one obvious next action.

Avoid:
- multiple competing CTAs,
- dense menus,
- overloaded dashboards,
- complex settings,
- long educational text.

### 2. The craving button is always easy to find

The user should never have to search for the main action.

Primary CTA:

> “Mam craving teraz”

This should be available from the home screen and ideally reachable quickly from key screens.

### 3. Short copy beats motivational speeches

Use short, concrete, emotionally safe language.

Good:

> “Tylko 60 sekund. Nie cały dzień.”

Bad:

> “You are on a transformative journey toward the best version of yourself.”

### 4. Relapse is not failure

No shame-based UI.

Do not:
- flash red failure states,
- reset progress harshly,
- say “you failed,”
- punish the user,
- over-celebrate perfection.

Use:

> “Slip to dane. Nie porażka.”

### 5. Default choices, not configuration

During MVP, prefer predefined options:
- trigger chips,
- intensity chips,
- action chips,
- simple status options.

Avoid free-text as the main input. Free-text can be optional.

### 6. Fast logging

Trigger logging should take less than 10–15 seconds.

If it takes longer, it will fail in the real moment.

### 7. Calm visual hierarchy

The UI should feel:
- spacious,
- calm,
- soft,
- adult,
- clear.

Avoid:
- aggressive red,
- casino-like gamification,
- childish badges,
- overly medical UI,
- fear-based imagery.

### 8. No guilt notifications

Notifications should be supportive, optional, and sparse.

Never:
- shame,
- pressure,
- manipulate,
- imply failure,
- create anxiety.

### 9. Progress means learning

Progress view should emphasize:
- patterns,
- recovery,
- cravings handled,
- helpful actions,
- returning after slips.

Not only:
- streak length,
- money saved,
- “perfect days.”

### 10. Reduce decisions during craving

In craving mode, the user should not choose from 20 strategies.

The app should suggest one action and allow “Daj inną.”

## Tone of voice

### Personality

- calm,
- direct,
- mature,
- warm,
- nonjudgmental,
- concise,
- practical.

### Avoid

- toxic positivity,
- moralizing,
- childish stickers,
- heroic language,
- medical certainty,
- “addiction guru” tone,
- long psychoeducation.

## Copywriting rules

### Use

- “możesz”
- “spróbuj”
- “jedna mała akcja”
- “to nadal się liczy”
- “zapiszmy wzorzec”
- “wróć od teraz”
- “bez oceniania”

### Avoid

- “musisz”
- “zawiodłeś”
- “przegrałeś”
- “zacznij od zera”
- “brak silnej woli”
- “ostatnia szansa”
- “nigdy więcej”

## Example screens

### Home

Title:

> “Co teraz?”

Primary CTA:

> “Mam craving teraz”

Secondary:
- “Miałem slip”
- “Dzienny check-in”
- “Moje triggery”
- “Postęp”

Small supportive copy:

> “Nie musisz ogarniać całego dnia. Wybierz jedną małą akcję.”

### Craving start

Title:

> “Zatrzymajmy automat.”

Body:

> “Tylko 60 sekund. Craving może być obecny, ale nie musi prowadzić decyzji.”

Primary CTA:

> “Pokaż małą akcję”

### Micro-action

Title:

> “Zrób teraz”

Action:

> “Wypij wodę i zmień miejsce na 2 minuty.”

Buttons:
- “Robię”
- “Daj inną”
- “Nie teraz”

### Trigger log

Title:

> “Co odpaliło craving?”

Options:
- Stres
- Nuda
- Alkohol
- Po jedzeniu
- Praca
- Towarzystwo
- Emocje
- Rano
- Inne

Small copy:

> “Jedno kliknięcie wystarczy.”

### Slip flow

Title:

> “To był slip, nie porażka.”

Body:

> “Zapiszmy tylko, co go uruchomiło. To pomoże następnym razem.”

Primary CTA:

> “Zapisz bez oceniania”

### Progress

Title:

> “Czego się uczysz?”

Cards:
- “Najczęstszy trigger”
- “Najbardziej pomocna akcja”
- “Cravingi zapisane”
- “Powroty po slipie”

## Microcopy library

### Craving

- “Tylko następna minuta.”
- “Nie musisz wygrać całego dnia.”
- “Zmień miejsce, zanim podejmiesz decyzję.”
- “Craving jest falą. Dajmy jej chwilę opaść.”
- “Jedna mała akcja zamiast automatu.”

### Stress

- “Najpierw obniż napięcie. Decyzja za chwilę.”
- “Zrób pauzę przed reakcją.”
- “Nie rozwiązuj stresu nikotyną przez najbliższe 60 sekund.”

### Boredom

- “To może być nuda, nie potrzeba nikotyny.”
- “Daj rękom coś innego do roboty.”
- “Zmień bodziec na 2 minuty.”

### Alcohol/social trigger

- “Najpierw oddal się od kontekstu.”
- “Weź coś do ręki bez nikotyny.”
- “Zrób 2 minuty przerwy od grupy.”

### Slip

- “To był slip, nie reset człowieka.”
- “Nie musisz czekać do jutra.”
- “Wracamy od teraz.”
- “Zapiszmy wzorzec, bez oceniania.”
- “Jedna decyzja nie kasuje całego wysiłku.”

### Success

- “To się liczy.”
- “Zatrzymałeś automat.”
- “Masz jeden zapis więcej. To buduje mapę triggerów.”
- “Mała akcja wykonana.”
- “Następnym razem aplikacja będzie wiedzieć więcej.”

## Notification tone

### Good

> “Możliwy moment ryzyka. Jedna mała akcja wystarczy.”

> “30 sekund check-inu? Bez oceniania.”

> “Jeśli był slip, wróć od teraz — nie od jutra.”

### Bad

> “Nie zepsuj swojej serii!”

> “Nie poddawaj się teraz!”

> “Pamiętaj, ile już straciłeś przez palenie.”

> “Zawalisz, jeśli nie wejdziesz do aplikacji.”

## Accessibility principles

MVP should support:

- large readable text,
- clear contrast,
- dynamic type where possible,
- simple tap targets,
- no interaction dependent only on color,
- minimal animation,
- no flashing or aggressive visual feedback.

## Things we must not do

Do not build:

- guilt-based streak reset,
- large onboarding quiz,
- medical scoring,
- dependence diagnosis,
- medication recommendation,
- complex habit dashboard,
- social pressure mechanics,
- public sharing,
- “leaderboards,”
- fear-based health imagery,
- long neuroscience lessons,
- notification spam,
- manipulative paywall during relapse/craving.

## UX acceptance test

Before accepting a screen, ask:

1. Can a stressed user understand this in 3 seconds?
2. Is there one clear next action?
3. Can the user skip without shame?
4. Is the copy medically safe?
5. Does this reduce cognitive load?
6. Does this help with craving, relapse recovery, or trigger intelligence?
7. Can this be simpler?
